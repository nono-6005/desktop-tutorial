# ファイル保存ルール

## 保存先
- Googleドライブのみ（ローカル保存しない）
- GAS Web App経由でGoogleドライブに送信する

## ファイル名
- analysis_YYYYMMDD_001.md　（複数枚の場合は連番）
- posts_YYYYMMDD_001.js

## 保存タイミング
- 分析完了後に自動でGASに送信する
- 複数画像の場合は1枚ごとに別送信する

# 分析軸の定義

## hookType（1つ選択）
数字型 / 逆張り型 / 共感型 / 断定型 / 疑問型 / 煽り型 / 物語型

## structure（1つ選択）
主張→根拠→事例 / 問題→解決→未来 / 体験談→学び→行動 /
常識の否定→真実→提案 / 質問→回答→深掘り / その他

## cta（1つ選択）
プロフ誘導 / リンク誘導 / リプ誘導 / 保存誘導 / なし

## readRetain[].type（各ワードに1つ）
禁断感 / 数字根拠 / 問いかけ / 対比 / 自己開示 /
未完結 / 感情語 / 権威性 / リスト予告 / 時間限定

# スコア採点基準（/50）

| 項目         | 満点 | 採点ポイント                     |
|--------------|------|----------------------------------|
| フックの引力 |  15  | 1行目で続きを読みたくなるか      |
| 完読維持力   |  10  | 離脱防止ワードの質・量           |
| 構造の明快さ |  10  | 起承転結・論理の流れ             |
| CTAの自然さ  |   8  | 行動喚起が押しつけがましくないか |
| 転用可能性   |   7  | 他ジャンルでも使えるテクニックか |

スコアは厳しめに採点する。インフレ禁止。

# 出力ルール

1. 必ず【出力①】→【出力②】の順で両方出力しGoogleドライブに送信する
2. 出力①と②は別ファイルに保存する
3. hook は投稿1行目テキストを完全一致で抜き出す（要約・改変禁止）
4. readRetain は最低1個・最大5個
5. stealPoints は転用可能性が高いものから最低1個・最大5個
6. tips・reason・effect・リズム分析・バズ仮説はすべて高校生にもわかる平易な言葉で書く
7. 心理学・行動経済学・脳科学の視点はClaude判断で必要な箇所に自然に組み込む
8. 既読本フレームワーク照合は全6冊すべてについて記載する
9. 複数画像の場合は画像ごとにNo.を振って両出力をセットで別ファイルに保存する
10. 送信完了後に「✅ Googleドライブ保存完了」とファイル名を表示する
